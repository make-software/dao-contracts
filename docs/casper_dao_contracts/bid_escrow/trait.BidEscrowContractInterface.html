Worker submits a Bid for a Job.

Parameters:
  • time - proposed timeframe for completing a Job
  • payment - proposed payment for a Job
  • reputation_stake - reputation stake for a Job if Worker is an Internal Worker
  • onboard - if Worker is an External Worker, then Worker can request to be onboarded after completing a Job
  • purse: purse containing stake from External Worker

fn pick_bid(&mut self, job_offer_id: u32, bid_id: u32, purse: URef)

Job poster picks a bid. This creates a new Job object and saves it in a storage. If worker is not onboarded, the job is accepted automatically. Otherwise, worker needs to accept job.

Events
  • JobCreated
  • JobAccepted
Errors
  • CannotPostJobForSelf when trying to create job for self.
  • JobPosterNotKycd or Error::WorkerNotKycd When either Job Poster or Worker has not completed the KYC process
source

fn submit_job_proof(&mut self, job_id: JobId, proof: DocumentHash)

Submits a job proof. This is called by a Worker or any KYC’d user during Grace Period. This starts a new voting over the result.

Events
  • JobSubmitted
Errors

Throws JobAlreadySubmitted if job was already submitted. Throws NotAuthorizedToSubmitResult if one of the constraints for job submission is not met.

Updates the old Bid and Job, the job is assigned to a new Worker. The rest goes the same as regular proof submission. See submit_job_proof(). The old Worker who didn’t submit the proof in time, is getting slashed.
